# OctoAcme Project Management Docs

Welcome! This README is the entry point for all core OctoAcme project management practices and provides links to detailed guides.

## Overview

OctoAcme delivers value through an iterative, outcome-focused, and transparent approach to project management. Work begins with a project one-pager that validates the idea, defines success metrics, aligns stakeholders, and surfaces risks before any planning begins. Once approved, the team breaks work into shippable increments with clear acceptance criteria, a Definition of Done, effort estimates, and a release plan — creating a lightweight but structured path from idea to delivery.

Roles are clearly defined across Project Managers, Product Managers, Developers, QA/Testing, and Stakeholders. The Product Manager owns the product vision, priorities, and success measurement, while the Project Manager coordinates delivery, schedules, risks, documentation, and communication. Developers are responsible for implementation and code reviews, and QA validates acceptance criteria. Stakeholders provide input and approvals at key milestones to keep work aligned with business outcomes.

Day-to-day execution is managed through regular ceremonies and visible workflow tooling. Daily standups, weekly delivery syncs, and sprint or milestone demos surface progress, blockers, dependencies, and risks. Work moves through a shared project board — Backlog → Ready → In Progress → In Review → QA → Done — supported by weekly PM/PdM alignment, periodic stakeholder updates, and escalation paths from the team to the PM, Product Lead, and Sponsor as needed.

Quality assurance and continuous improvement are built into both delivery and release practices. The documentation emphasises small pull requests, linked issues, automated tests and linting in CI, and at least one approval before merging. Releases require passing acceptance criteria, CI, and security scans, plus prepared release notes and documented rollback plans. After each sprint, release, or incident, retrospectives capture wins, improvement areas, and concrete action items that feed back into the backlog to continuously raise the bar.
