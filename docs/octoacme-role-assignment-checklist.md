# OctoAcme — Role Assignment Checklist

## Purpose
Ensure every project has named owners for each role before moving from Initiation into Planning. Use this checklist during the project kickoff to reduce ambiguity and prevent gaps in accountability.

---

## How to use
1. Complete this checklist as part of the [Project Initiation](./octoacme-project-initiation.md) phase.
2. Record the assigned person (or "TBD + date needed by") for each role.
3. Attach or link the completed checklist to the Project One-pager or project board.
4. Review and update at the start of each planning cycle when team composition changes.

## Role Assignment Checklist

### Initiation
- [ ] Project Manager named and confirmed available
- [ ] Product Manager named and product vision confirmed
- [ ] Stakeholder sponsor identified and engaged
- [ ] Business Analyst assigned to lead requirements gathering
- [ ] UX Designer engaged if user-facing work is in scope
- [ ] All role assignments documented in the Project One-pager

### Planning
- [ ] Technical Lead confirmed for kickoff and design reviews
- [ ] QA Lead engaged to review and accept test strategy
- [ ] Scrum Master confirmed to facilitate planning and sprint ceremonies
- [ ] Technical Writer briefed on feature scope and milestone dates
- [ ] DevOps Engineer briefed on environment and deployment requirements
- [ ] Any third-party or shared service owners identified and contacted

### Execution
- [ ] All roles have access to the project board and relevant repos
- [ ] On-call / escalation contacts documented and shared with the team
- [ ] Backup owners identified for critical roles (vacation/absence cover)
- [ ] Role-specific contacts documented in the project README or comms plan

### Release
- [ ] QA sign-off owner confirmed
- [ ] DevOps deployment owner confirmed for release window
- [ ] Technical Writer confirmed for final release notes review
- [ ] Stakeholder communication owner confirmed (usually PM or Product Manager)
- [ ] Post-release monitoring owner assigned
